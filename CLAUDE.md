# Stewart Masters — Site Guidelines

## Blog post writing rules

All blog posts must read as written by a human operator with 20 years of direct experience. Google and AI detection tools flag specific patterns. Avoid every one of the following, no exceptions.

### Banned phrases and words

Never use these — not once, not in any variation:

- "it's worth noting" / "it is worth noting"
- "it's important to note" / "it is important to note"
- "delve into" / "delves into" / "dive deep"
- "in conclusion" / "to summarise" / "to conclude"
- "Furthermore," / "Moreover," / "Additionally," (as sentence openers)
- "cutting-edge"
- "game-changing" / "game-changer"
- "transformative" / "transformation" overuse (once per post max)
- "leverage" as a verb (use "use" instead)
- "robust" (use "solid", "reliable", "strong")
- "streamline" (be specific: "cut the steps", "reduce the handoffs")
- "landscape" as metaphor ("AI landscape" → "AI market", "the field")
- "navigate" as metaphor ("navigate complexity" → "deal with", "handle")
- "seamless" / "seamlessly"
- "ecosystem" when used loosely
- "deep dive"
- "unpack"
- "needless to say"
- "at the end of the day"
- "in today's fast-paced world" or any variation

### Em dashes

Em dashes (—) are the single strongest AI signal. Use a maximum of **2 per article**, and only where nothing else works. In most cases, use a comma, a period, or rewrite the sentence.

Wrong: "Most AI projects fail — not because of the model, but the integration layer."
Right: "Most AI projects fail. Not because of the model. Because of the integration layer."

Wrong: "The team built a system — and then never used it."
Right: "The team built a system they never used."

### Structure tells to avoid

- Don't write sections of exactly equal length
- Don't end every section with a tidy three-word takeaway
- Don't use bullet lists for everything — prose is fine, use bullets only when a list is genuinely a list
- Don't open paragraphs with "This means that..."
- Don't close articles with a call-to-action paragraph that sounds like an ad

### Voice to aim for

The writing should sound like Stewart talking to a senior executive at a dinner: direct, a bit impatient with bullshit, grounded in what he's actually seen go wrong. Short sentences. Specific examples over abstract principles. No throat-clearing. Start with the point, not the setup.

### What good looks like

- Sentences of mixed length — some short and punchy, some longer and more complex
- Opinions stated as opinions, not hedged into oblivion
- Specific numbers and situations where possible
- The occasional incomplete sentence for emphasis. Like this.
- First-person perspective: "I've seen this fail" not "organizations often find"

## Stack & deploy

- **Static HTML** — no framework, no build framework
- **CSS**: Tailwind (built via `npm run build:css` → `styles.css`)
- **Deploy**: Netlify, publish dir is `.` (root), push to `main` deploys automatically
- **Build command**: `npm ci && npm run build:css`
- **Push**: `git add`, `git commit`, `git pull --rebase`, `git push`

## Automated blog publishing

Blog posts are scheduled via `blog/schedule.json`. Publishing is automated via GitHub Actions running `publish.py` daily at 08:00 UTC (09:00 CET):
1. Moves post HTML from `blog/drafts/` → `blog/`
2. Updates `blog/index.html` featured section
3. Regenerates `feed.xml`

Drafts in `blog/drafts/` are blocked from public access via netlify.toml redirect (404).

## Content structure

```
index.html              # Homepage
blog/
  index.html            # Blog listing
  schedule.json         # Scheduled publish dates
  drafts/               # Future posts (not publicly accessible)
  {slug}.html           # Published posts
speaking.html           # Speaking page (English)
ponente.html            # Speaking page (Spanish)
speaking-topics.json    # Speaking topic data (read by generate_speaking.py)
contact.html
glossary.html
faq/                    # FAQ pages (generated by publish_faq.py)
styles.css              # Built Tailwind output
tailwind.config.js
```

## Scripts

```bash
npm run build:css       # Rebuild Tailwind CSS (run after changing HTML classes)
python3 publish.py      # Publish today's scheduled post
python3 generate_speaking.py   # Regenerate speaking page from speaking-topics.json
python3 publish_faq.py  # Regenerate FAQ pages
python3 sanitize_posts.py      # Sanitize blog post HTML
```
